# 网吧管理系统接口文档

> 说明：本文件作为后续接口设计的统一入口，新增接口请按章节追加在本文件中。

## 1. 通用约定

### 1.1 统一响应结构

```json
{
  "code": 0,
  "message": "OK",
  "data": {}
}
```


- code：0 表示成功，非 0 表示失败
- message：提示信息
- data：业务数据

### 1.2 鉴权方式

- 采用 Bearer Token
- Header：`Authorization: Bearer <token>`

## 2. 认证

### 2.1 管理员登录

- **接口**：`POST /auth/login`
- **描述**：管理员账号密码登录
- **鉴权**：无

**请求体**
```json
{
  "username": "admin",
  "password": "123456"
}
```

**curl 示例**
```bash
curl -X POST "http://localhost:8080/auth/login" \
  -H "Content-Type: application/json" \
  -d '{
    "username": "admin",
    "password": "123456"
  }'
```

**成功响应**
```json
{
  "code": 0,
  "message": "OK",
  "data": {
    "token": "<jwt-token>",
    "tokenType": "Bearer",
    "expiresIn": 7200,
    "role": "SUPER_ADMIN",
    "admin": {
      "id": 1,
      "username": "admin",
      "name": "张三",
      "role": "SUPER_ADMIN"
    }
  }
}
```

**失败响应示例**
```json
{
  "code": 40101,
  "message": "账号或密码错误",
  "data": null
}
```

**业务规则**
- 账号或密码错误返回失败
- 可选：连续 5 次失败锁定 10 分钟（前端已有提示）

### 2.2 管理员登出

- **接口**：`POST /auth/logout`
- **描述**：管理员退出登录，服务端清理会话或拉黑 token
- **鉴权**：需要

**请求头**
```
Authorization: Bearer <token>
```

**请求体**
```json
{}
```

**成功响应**
```json
{
  "code": 0,
  "message": "OK",
  "data": null
}
```

**失败响应示例**
```json
{
  "code": 401,
  "message": "未登录或登录已过期",
  "data": null
}
```

## 3. 员工与角色管理

> 说明：该模块仅超级管理员可用，用于管理员账号与角色的基础管理。当前为 Demo 版本，不做复杂 RBAC，只支持 `SUPER_ADMIN` 和 `ADMIN` 两种后台角色。

### 3.2 分页查询员工列表

- **接口**：`GET /admins`
- **描述**：分页查询管理员账号列表
- **鉴权**：需要，且仅 `SUPER_ADMIN`

**查询参数**

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| keyword | string | 否 | 按账号或姓名做前缀查询，便于列表筛选 |
| role | string | 否 | 角色编码：`SUPER_ADMIN` / `ADMIN` |
| status | int | 否 | 状态：`1` 启用，`0` 禁用 |
| page | int | 否 | 页码，默认 `1` |
| size | int | 否 | 每页条数，默认 `20` |

**curl 示例**
```bash
curl "http://localhost:8080/admins?keyword=admin&role=ADMIN&status=1&page=1&size=10" \
  -H "Authorization: Bearer <token>"
```

**成功响应**
```json
{
  "code": 0,
  "message": "OK",
  "data": {
    "total": 2,
    "page": 1,
    "size": 10,
    "items": [
      {
        "id": 5,
        "username": "admin_test1",
        "name": "测试管理员1",
        "role": "ADMIN",
        "roleLabel": "管理员",
        "status": 1,
        "statusLabel": "启用",
        "lastLoginTime": "2026-03-23T21:00:00",
        "createdAt": "2026-03-20T10:00:00",
        "updatedAt": "2026-03-23T21:00:00"
      }
    ]
  }
}
```

**业务规则**
- `keyword` 为账号/姓名的前缀查询，适合 Demo 列表筛选
- 系统至少保留一个启用中的超级管理员

### 3.3 新增员工

## 11. 运营统计

> 说明：该模块仅超级管理员可见，当前为 Demo 版本，统计口径按“订单开始时间 `session_order.start_time`”和“充值时间 `recharge_record.created_at`”归属到统计区间，用于保证查询命中索引。

### 11.1 统计总览

- **接口**：`GET /stats/overview`
- **描述**：返回统计卡片数据
- **鉴权**：需要，且仅 `SUPER_ADMIN`

**查询参数**

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| start | string | 否 | 开始日期，格式 `YYYY-MM-DD`，默认当月第一天 |
| end | string | 否 | 结束日期，格式 `YYYY-MM-DD`，默认今天 |

**成功响应**
```json
{
  "code": 0,
  "message": "OK",
  "data": {
    "startDate": "2026-03-01",
    "endDate": "2026-03-25",
    "cashFlowAmount": 1975.00,
    "cashFlowLabel": "¥1975.00",
    "sessionRevenueAmount": 268.10,
    "sessionRevenueLabel": "¥268.10",
    "totalDurationMinutes": 2395,
    "totalDurationLabel": "2395 分钟",
    "activeUsers": 14,
    "activeUsersLabel": "14 人",
    "arpu": 19.15,
    "arpuLabel": "¥19.15",
    "peakHour": "18:00"
  }
}
```

**业务规则**
- `cashFlowAmount` 统计充值流水
- `sessionRevenueAmount` 统计上机消费金额
- `activeUsers` 统计区间内有上机或充值行为的去重用户数
- `ARPU = sessionRevenueAmount / activeUsers`
- 默认统计区间不能超过 `366` 天

**成功响应**
```json
{
  "code": 0,
  "message": "OK",
  "data": {
    "machineRevenueTop": [
      { "machineId": 4002, "machineCode": "F-201", "revenueAmount": 51.24, "revenueLabel": "¥51.24", "sessionCount": 1 }
    ],
    "userConsumeTop": [

  }

```

**查询与性能说明**
- 上机统计统一按 `session_order.start_time` 走时间范围过滤，命中 `idx_session_status_start`
- 充值统计统一按 `recharge_record.created_at` 走时间范围过滤，命中 `idx_recharge_created`
- 低余额列表按 `user.status` 过滤，命中 `idx_user_status_balance`

**业务规则**
- `username` 不能重复
- `role` 仅支持 `SUPER_ADMIN`、`ADMIN`
- `status` 仅支持 `1`、`0`，不传默认 `1`

## 6. 上机管理

> 说明：该模块面向超级管理员和管理员，当前为 Demo 版本，目标是支撑进行中订单、暂停锁定订单、历史订单、开通上机和强制下机。

### 6.1 开通上机

- **接口**：`POST /sessions/open`
- **描述**：为指定用户和机位创建进行中的上机订单
- **鉴权**：需要，`SUPER_ADMIN` / `ADMIN`

**请求体**
```json
{
  "userId": 4,
  "machineId": 1
}
```

**业务规则**
- 用户必须存在且为可用状态
- 机位必须存在且为“空闲”状态
