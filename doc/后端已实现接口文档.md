# 网吧管理系统接口文档

> 说明：本文件作为后续接口设计的统一入口，新增接口请按章节追加在本文件中。

## 1. 通用约定

### 1.1 统一响应结构

```json
{
  "code": 0,
  "message": "OK",
  "data": {}
}
```


- code：0 表示成功，非 0 表示失败
- message：提示信息
- data：业务数据

### 1.2 鉴权方式

- 采用 Bearer Token
- Header：`Authorization: Bearer <token>`

## 2. 认证

### 2.1 管理员登录

- **接口**：`POST /auth/login`
- **描述**：管理员账号密码登录
- **鉴权**：无

**请求体**
```json
{
  "username": "admin",
  "password": "123456"
}
```

**curl 示例**
```bash
curl -X POST "http://localhost:8080/auth/login" \
  -H "Content-Type: application/json" \
  -d '{
    "username": "admin",
    "password": "123456"
  }'
```

**成功响应**
```json
{
  "code": 0,
  "message": "OK",
  "data": {
    "token": "<jwt-token>",
    "tokenType": "Bearer",
    "expiresIn": 7200,
    "role": "SUPER_ADMIN",
    "admin": {
      "id": 1,
      "username": "admin",
      "name": "张三",
      "role": "SUPER_ADMIN"
    }
  }
}
```

**失败响应示例**
```json
{
  "code": 40101,
  "message": "账号或密码错误",
  "data": null
}
```

**业务规则**
- 账号或密码错误返回失败
- 可选：连续 5 次失败锁定 10 分钟（前端已有提示）

### 2.2 管理员登出

- **接口**：`POST /auth/logout`
- **描述**：管理员退出登录，服务端清理会话或拉黑 token
- **鉴权**：需要

**请求头**
```
Authorization: Bearer <token>
```

**请求体**
```json
{}
```

**成功响应**
```json
{
  "code": 0,
  "message": "OK",
  "data": null
}
```

**失败响应示例**
```json
{
  "code": 401,
  "message": "未登录或登录已过期",
  "data": null
}
```

## 3. 员工与角色管理

> 说明：该模块仅超级管理员可用，用于管理员账号与角色的基础管理。当前为 Demo 版本，不做复杂 RBAC，只支持 `SUPER_ADMIN` 和 `ADMIN` 两种后台角色。

### 3.1 获取角色选项

- **接口**：`GET /admins/role-options`
- **描述**：获取管理员角色下拉选项
- **鉴权**：需要，且仅 `SUPER_ADMIN`

**成功响应**
```json
{
  "code": 0,
  "message": "OK",
  "data": [
    {
      "code": "SUPER_ADMIN",
      "label": "超级管理员",
      "description": "拥有系统全部权限，可管理员工账号与系统配置"
    },
    {
      "code": "ADMIN",
      "label": "管理员",
      "description": "负责用户、机位和订单等日常运营管理"
    }
  ]
}
```

### 3.2 分页查询员工列表

- **接口**：`GET /admins`
- **描述**：分页查询管理员账号列表
- **鉴权**：需要，且仅 `SUPER_ADMIN`

**查询参数**

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| keyword | string | 否 | 按账号或姓名做前缀查询，便于列表筛选 |
| role | string | 否 | 角色编码：`SUPER_ADMIN` / `ADMIN` |
| status | int | 否 | 状态：`1` 启用，`0` 禁用 |
| page | int | 否 | 页码，默认 `1` |
| size | int | 否 | 每页条数，默认 `20` |

**curl 示例**
```bash
curl "http://localhost:8080/admins?keyword=admin&role=ADMIN&status=1&page=1&size=10" \
  -H "Authorization: Bearer <token>"
```

**成功响应**
```json
{
  "code": 0,
  "message": "OK",
  "data": {
    "total": 2,
    "page": 1,
    "size": 10,
    "items": [
      {
        "id": 5,
        "username": "admin_test1",
        "name": "测试管理员1",
        "role": "ADMIN",
        "roleLabel": "管理员",
        "status": 1,
        "statusLabel": "启用",
        "lastLoginTime": "2026-03-23T21:00:00",
        "createdAt": "2026-03-20T10:00:00",
        "updatedAt": "2026-03-23T21:00:00"
      }
    ]
  }
}
```

**业务规则**
- `keyword` 为账号/姓名的前缀查询，适合 Demo 列表筛选
- 系统至少保留一个启用中的超级管理员

### 3.3 新增员工

- **描述**：新增管理员账号
- **鉴权**：需要，且仅 `SUPER_ADMIN`

## 6. 上机管理

> 说明：该模块面向超级管理员和管理员，当前为 Demo 版本，目标是支撑进行中订单、历史订单、开通上机和强制下机。

### 6.1 开通上机

- **接口**：`POST /sessions/open`
- **描述**：为指定用户和机位创建进行中的上机订单
- **鉴权**：需要，`SUPER_ADMIN` / `ADMIN`

**请求体**
```json
{
  "userId": 4,
  "machineId": 1
}
```

**业务规则**
- 用户必须存在且为可用状态
- 机位必须存在且为“空闲”状态
- 用户余额必须大于等于机位单价
- 同一用户、同一机位都不能存在另一条进行中的订单
- 开通成功后，机位状态自动切换为“使用中”

### 6.2 强制下机

- **接口**：`PUT /sessions/{id}/force-end`
- **描述**：管理员强制结束进行中的订单
- **鉴权**：需要，`SUPER_ADMIN` / `ADMIN`

**业务规则**
- 仅进行中的订单可以强制下机
- 强制下机后订单状态改为 `2`
- 机位状态自动回到“空闲”
- 订单金额按当前已记录分钟数结算；若用户余额不足，则最多扣至 `0`

### 6.3 分页查询进行中订单

- **接口**：`GET /sessions/current`
- **描述**：分页查询进行中的实时订单
- **鉴权**：需要，`SUPER_ADMIN` / `ADMIN`

**查询参数**

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| keyword | string | 否 | 支持按用户姓名、手机号、身份证或机位编号前缀搜索 |
| page | int | 否 | 页码，默认 `1` |
| size | int | 否 | 每页条数，默认 `20` |

**成功响应**
```json
{
  "code": 0,
  "message": "OK",
  "data": {
    "total": 1,
    "page": 1,
    "size": 20,
```

### 6.4 分页查询历史订单

- **接口**：`GET /sessions/history`
- **描述**：分页查询已完成和强制结束订单
- **鉴权**：需要，`SUPER_ADMIN` / `ADMIN`

**查询参数**

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| keyword | string | 否 | 支持按用户姓名、手机号、身份证或机位编号前缀搜索 |
| status | int | 否 | 仅支持 `1` 完成、`2` 强制结束 |
| dateFrom | string | 否 | 开始日期，格式 `YYYY-MM-DD` |
| dateTo | string | 否 | 结束日期，格式 `YYYY-MM-DD` |
| page | int | 否 | 页码，默认 `1` |
| size | int | 否 | 每页条数，默认 `20` |

**业务规则**
- 历史订单默认查询状态 `1` 和 `2`
- 日期范围查询命中 `session_order.start_time` 索引
- 关键字先解析成 `user_id` / `machine_id`，再回查订单，避免直接扫订单表
