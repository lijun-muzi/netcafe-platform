# 网吧管理系统接口文档

> 说明：本文件作为后续接口设计的统一入口，新增接口请按章节追加在本文件中。

## 1. 通用约定

### 1.1 统一响应结构

```json
{
  "code": 0,
  "message": "OK",
  "data": {}
}
```

- code：0 表示成功，非 0 表示失败
- message：提示信息
- data：业务数据

### 1.2 鉴权方式

- 采用 Bearer Token
- Header：`Authorization: Bearer <token>`

## 2. 认证

### 2.1 管理员登录

- **接口**：`POST /auth/login`
- **描述**：管理员账号密码登录
- **鉴权**：无

**请求体**
```json
{
  "username": "admin",
  "password": "123456"
}
```

**curl 示例**
```bash
curl -X POST "http://localhost:8080/auth/login" \
  -H "Content-Type: application/json" \
  -d '{
    "username": "admin",
    "password": "123456"
  }'
```

**成功响应**
```json
{
  "code": 0,
  "message": "OK",
  "data": {
    "token": "<jwt-token>",
    "tokenType": "Bearer",
    "expiresIn": 7200,
    "role": "SUPER_ADMIN",
    "admin": {
      "id": 1,
      "username": "admin",
      "name": "张三",
      "role": "SUPER_ADMIN"
    }
  }
}
```

**失败响应示例**
```json
{
  "code": 40101,
  "message": "账号或密码错误",
  "data": null
}
```

**业务规则**
- 账号或密码错误返回失败
- 可选：连续 5 次失败锁定 10 分钟（前端已有提示）

### 2.2 管理员登出

- **接口**：`POST /auth/logout`
- **描述**：管理员退出登录，服务端清理会话或拉黑 token
- **鉴权**：需要

**请求头**
```
Authorization: Bearer <token>
```

**请求体**
```json
{}
```

**成功响应**
```json
{
  "code": 0,
  "message": "OK",
  "data": null
}
```

**失败响应示例**
```json
{
  "code": 401,
  "message": "未登录或登录已过期",
  "data": null
}
```
