# 网吧管理系统方案设计

## 0. 版本记录
- v1.0（2026-01-29）：基于需求文档的简化版方案，计费采用方案A（后台每分钟扣费）。

## 0.1 设计范围（MVP）
- 仅覆盖单网吧场景，角色：超级管理员/管理员/用户。
- 不包含：多门店、线上支付、短信/密码登录、复杂运营报表。
- 目标：实现“能用、够用、可持续迭代”的基础网吧管理系统。

## 1. 技术栈与框架
- 后端：Spring Boot + Spring Security + MyBatis-Plus + Maven，参考若依后端分层与安全框架。
- 前端：Vue 3，参考若依 Vue 版项目结构（路由/菜单/权限指令/axios 拦截器）。
- 数据库：MySQL（建议 8.x），分库分表暂不考虑。
- 日志与监控：Spring Boot Actuator、操作审计表；按需接入 ELK/Prometheus（可选）。

## 2. 总体架构
- 前后端分离：管理端（超级管理员/管理员），用户端（自助上机）两个前端。
- 后端分层：Controller → Service → Domain/Mapper；DTO/VO 分离；使用统一响应体与全局异常处理。
- 安全：基于 Spring Security + JWT；RBAC 权限模型；后端路径与方法级鉴权。
- 部署：单体部署即可，Nginx 反向代理前端与后端 API。

## 3. 角色与权限模型
- 角色：SUPER_ADMIN、ADMIN、USER（用户端）。
- 权限（示例）：  
  - 员工管理：`staff:list/create/update/delete`（仅 SUPER_ADMIN）  
  - 价格配置：`machine:price:update`（仅 SUPER_ADMIN）  
  - 机位管理：`machine:list/create/update/delete/forceOffline/open`（SUPER_ADMIN/ADMIN）  
  - 用户管理：`user:list/create/update/delete/recharge/freeze`（SUPER_ADMIN/ADMIN）  
  - 运营统计：`stats:view`（SUPER_ADMIN）  
  - 用户端：`session:self`（USER）  
- 菜单与路由按角色分配；前端基于路由守卫与后端返回的角色/权限控制。

## 4. 核心业务设计
### 4.1 上/下机与计费
- 计费粒度：每分钟计费；机位单价默认 0.1 元/分钟，超级管理员可机位级配置。
- 上机流程：管理员在后台为用户分配机位 → 创建“进行中”订单，记录开始时间和单价快照 → 用户在机位端身份证登录校验订单可用 → 前端按服务端时间展示计时。
- 计费方案A（后台每分钟扣费）：  
  - 定时任务每分钟扫描“进行中”订单，按机位单价扣减余额并累加订单已计费分钟数。  
  - 当余额不足扣除下一分钟费用时，立即强制下机并结算到当前分钟。  
  - 余额低于 60 分钟费用时触发提醒（前端弹窗/提示）。  
- 下机流程：用户自助下机或管理员强制下机 → 结算到“已计费分钟” → 订单标记完成/强制结束。

### 4.2 账户与充值
- 充值方式：管理员线下录入，记录渠道（现金/其他）、经办人、备注。
- 身份证唯一约束；余额扣减通过事务保证原子性。

### 4.3 机位管理
- 状态：空闲/使用中/停用；停用需先结束当前订单。
- 设备配置以 JSON 存储，支持模板快速建机位（本地模板即可，无导入导出）。

### 4.4 运营统计
- 维度：月度/自定义区间的流水、上机时长、活跃用户、机位上机率、TOP/闲置机位、消费 TOP 用户。
- 实现：SQL 聚合或预计算汇总表（按日/月）+ 缓存。
- 无导出需求，前端图表展示。

## 5. 数据模型（建议）
- user（用户）：id，mobile(unique)，id_card(unique)，name，balance，status(enum:active/frozen)，register_time，last_login_time，last_session_time。
- admin（管理员）：id，username(unique)，password(hash)，name，role(SUPER_ADMIN/ADMIN)，status，last_login_time。
- machine（机位）：id，code(unique)，status(enum:idle/using/disabled)，price_per_min(decimal)，config_json，created_at，updated_at。
- session_order（上机订单）：id，user_id，machine_id，start_time，end_time，duration_minutes，price_snapshot，amount，billed_minutes，last_billed_time，status(enum:ongoing/finished/forced)，force_by_admin_id(nullable)。
- recharge_record：id，user_id，amount，channel，operator_admin_id，created_at，remark。
- audit_log：id，operator_id，operator_role，action，target_type，target_id，before_data，after_data，created_at。
- price_change_log（可选）：id，machine_id，old_price，new_price，operator_admin_id，created_at。
- 索引建议：user(id_card, mobile)，machine(code)，session_order(user_id, machine_id, start_time)，recharge_record(user_id, created_at)。查询需命中索引，避免全表扫描。

## 6. 前端设计要点（Vue3，参考若依）
- 目录：`src/views/admin/...`（管理端），`src/views/user/...`（用户端）；共享组件与接口封装。
- 鉴权：登录后存储 token，路由守卫校验角色；按钮级权限指令。
- 管理端页面：员工管理、用户管理、机位管理（含实时面板）、运营统计、系统设置。
- 用户端页面：登录（身份证）、当前机位与计时展示、续费入口提示（跳转管理员代充值提示）、下机按钮、余额提醒弹窗。
- 网络层：axios 拦截器加 token，统一错误提示；时间显示以服务端时间为准。

## 7. 后端设计要点
- 安全：Spring Security + JWT；密码加密存储；关键接口防重放（如充值、下机结算可加幂等号）。
- 事务：充值、下机结算需事务包裹，扣费与订单状态更新同事务。
- 余额扣减：使用版本号或 `for update` 防止并发超扣。
- 审计：操作日志 AOP（价格修改、强制下机、充值、用户状态变更等）。
- 全局异常与统一返回码：封装 `code/message/data`。
- 时间基准：以服务器时间为准，前端只展示。
- 定时任务（必需）：每分钟执行扣费（方案A），并在余额不足时自动强制下机与结算。

## 8. 典型接口设计（示例）
- 认证：`POST /auth/login`（管理员账号密码），`POST /auth/user-login`（用户身份证）。
- 员工管理（SUPER_ADMIN）：`GET/POST/PUT/DELETE /admins`，`PUT /admins/{id}/status`。
- 用户管理：`GET/POST/PUT/DELETE /users`，`POST /users/{id}/recharge`，`PUT /users/{id}/status`。
- 机位管理：`GET/POST/PUT/DELETE /machines`，`PUT /machines/{id}/price`（仅 SUPER_ADMIN），`PUT /machines/{id}/disable`，`PUT /machines/{id}/enable`。
- 上机控制：`POST /sessions/open`（admin 指定 user+machine），`PUT /sessions/{id}/force-end`（admin），`POST /sessions/user-end`（user 端自助下机）。
- 查询与监控：`GET /sessions/current`（实时列表），`GET /sessions/history`（过滤 user/machine/time）。
- 统计：`GET /stats/overview?start&end`，`GET /stats/machine-usage`，`GET /stats/user-consume-top`。
- 所有查询接口需走索引字段过滤（如时间范围 + user_id/machine_id），避免全表扫描。
 - 详细接口清单见第 12 章。

## 9. 配置与参数
- 默认单价：0.1 元/分钟；可机位级覆盖。
- 余额提醒阈值：60 分钟费用（可配置）。
- 订单计费：按分钟扣费（方案A）；金额=已计费分钟 * 单价。
- 会话超时：管理员端 30 分钟不操作需重新登录；用户端前端保持心跳，后端以订单状态为准。

## 10. 非功能设计
- 安全：RBAC、JWT、密码加密、操作审计、身份证脱敏展示。
- 可用性：计费核心路径优先保障；异常重试与兜底任务。
- 性能：统计接口使用聚合并限制时间范围；机位状态与进行中订单直接读写数据库，必要时可考虑本地缓存或后续引入缓存方案。
- 运维：分环境配置（dev/test/prod），日志分级与轮转；健康检查与告警。

## 11. 数据库表结构设计（MVP）
说明：以下为核心表结构草案，满足需求文档与计费方案A。字段可按实际框架规范微调。

### 11.1 user（用户）
- id BIGINT PK
- mobile VARCHAR(20) UNIQUE
- id_card VARCHAR(20) UNIQUE
- name VARCHAR(50)
- balance DECIMAL(10,2) DEFAULT 0.00
- status TINYINT DEFAULT 1（1可用/0冻结）
- register_time DATETIME
- last_login_time DATETIME NULL
- last_session_time DATETIME NULL
- created_at DATETIME
- updated_at DATETIME
- 索引：UNIQUE(mobile)、UNIQUE(id_card)、IDX(status)

### 11.2 admin（管理员）
- id BIGINT PK
- username VARCHAR(50) UNIQUE
- password VARCHAR(255)
- name VARCHAR(50)
- role VARCHAR(20)（SUPER_ADMIN/ADMIN）
- status TINYINT DEFAULT 1
- last_login_time DATETIME NULL
- created_at DATETIME
- updated_at DATETIME
- 索引：UNIQUE(username)、IDX(status)

### 11.3 machine（机位）
- id BIGINT PK
- code VARCHAR(32) UNIQUE
- status TINYINT（0空闲/1使用中/2停用）
- price_per_min DECIMAL(10,4) DEFAULT 0.1000
- config_json JSON
- template_id BIGINT NULL（来自模板）
- last_maintained_at DATETIME NULL
- created_at DATETIME
- updated_at DATETIME
- 索引：UNIQUE(code)、IDX(status)

### 11.4 machine_template（机位模板）
- id BIGINT PK
- name VARCHAR(50)
- config_json JSON
- created_at DATETIME
- updated_at DATETIME
- 索引：UNIQUE(name)

### 11.5 session_order（上机订单）
- id BIGINT PK
- user_id BIGINT
- machine_id BIGINT
- start_time DATETIME
- end_time DATETIME NULL
- duration_minutes INT DEFAULT 0
- price_snapshot DECIMAL(10,4)
- amount DECIMAL(10,2) DEFAULT 0.00
- billed_minutes INT DEFAULT 0
- last_billed_time DATETIME NULL
- status TINYINT（0进行中/1完成/2强制结束）
- force_by_admin_id BIGINT NULL
- created_at DATETIME
- updated_at DATETIME
- 索引：IDX(status)、IDX(user_id, start_time)、IDX(machine_id, start_time)

### 11.6 recharge_record（充值记录）
- id BIGINT PK
- user_id BIGINT
- amount DECIMAL(10,2)
- channel VARCHAR(20)
- operator_admin_id BIGINT
- remark VARCHAR(255) NULL
- created_at DATETIME
- 索引：IDX(user_id, created_at)、IDX(created_at)

### 11.7 audit_log（审计日志）
- id BIGINT PK
- operator_id BIGINT
- operator_role VARCHAR(20)
- action VARCHAR(50)
- target_type VARCHAR(50)
- target_id BIGINT
- before_data JSON NULL
- after_data JSON NULL
- created_at DATETIME
- 索引：IDX(created_at)、IDX(operator_id)、IDX(target_type)

### 11.8 system_config（系统配置）
- id BIGINT PK
- config_key VARCHAR(50) UNIQUE（如 balance_warning_minutes, default_price_per_min）
- config_value VARCHAR(255)
- description VARCHAR(255) NULL
- updated_at DATETIME

### 11.9 price_change_log（可选）
- id BIGINT PK
- machine_id BIGINT
- old_price DECIMAL(10,4)
- new_price DECIMAL(10,4)
- operator_admin_id BIGINT
- created_at DATETIME
- 索引：IDX(machine_id, created_at)

## 12. 接口清单（MVP）
说明：仅列核心接口，均需鉴权；查询接口默认分页，且需通过索引字段过滤。

### 12.1 认证
- POST /auth/login（管理员登录）
- POST /auth/user-login（用户身份证登录）
- POST /auth/logout（退出）

### 12.2 员工管理（SUPER_ADMIN）
- GET /admins（分页查询）
- POST /admins（新增）
- PUT /admins/{id}（编辑）
- PUT /admins/{id}/status（启用/禁用）
- PUT /admins/{id}/password-reset（重置密码）
- DELETE /admins/{id}（删除）

### 12.3 用户管理（SUPER_ADMIN/ADMIN）
- GET /users（按手机号/身份证/姓名/状态）
- POST /users（新增用户档案）
- PUT /users/{id}（编辑）
- PUT /users/{id}/status（冻结/解冻）
- POST /users/{id}/recharge（充值）
- GET /users/{id}/recharges（充值记录）
- GET /users/{id}/sessions（上机订单记录）

### 12.4 机位管理（SUPER_ADMIN/ADMIN）
- GET /machines（按状态/价格/配置标签）
- POST /machines（新增）
- PUT /machines/{id}（编辑）
- PUT /machines/{id}/status（启用/停用）
- PUT /machines/{id}/price（仅SUPER_ADMIN）
- POST /machines/batch-create（模板批量建机位）
- GET /machines/monitor（实时面板）

### 12.5 机位模板（SUPER_ADMIN/ADMIN）
- GET /machine-templates
- POST /machine-templates
- PUT /machine-templates/{id}
- DELETE /machine-templates/{id}

### 12.6 上机/下机与订单
- POST /sessions/open（管理员为用户开通上机）
- PUT /sessions/{id}/force-end（管理员强制下机）
- POST /sessions/user-end（用户自助下机）
- GET /sessions/current（进行中订单列表）
- GET /sessions/history（订单历史，需时间范围）

### 12.7 充值与对账
- GET /recharges（按用户/时间范围/渠道）

### 12.8 运营统计（SUPER_ADMIN）
- GET /stats/overview?start&end（流水/时长/活跃用户/ARPU）
- GET /stats/machine-usage?start&end（机位上机率）
- GET /stats/machine-top?start&end（收入TOP机位）
- GET /stats/idle-machines?start&end（长时间闲置）
- GET /stats/user-top?start&end（消费TOP用户）
- GET /stats/low-balance?threshold（低余额清单）
- GET /stats/overview/export?format=csv（导出CSV）

### 12.9 系统设置（SUPER_ADMIN）
- GET /system/config
- PUT /system/config

### 12.10 审计日志（SUPER_ADMIN）
- GET /audit/logs（按操作人/类型/时间范围）

## 13. 风险与待确认
- 支付：当前仅线下管理员录入，后续若接入线上需补充支付回调与对账。
- 身份验证：用户端仅身份证，是否需二次校验（短信/密码）待确认。
- 计费精度：当前按分钟，若需按秒需调整金额计算与存储精度。

## 14. 里程碑建议
- M1：基础用户/管理员/机位 CRUD + 登录鉴权。
- M2：上机/下机全链路与计费结算，余额控制。
- M3：运营统计与实时面板，审计日志。
- M4：优化与验收（性能、异常兜底、UI 打磨）。
