# 前端接口需求清单

> 说明：本文件用于记录前端开发过程中已经遇到、且后端当前接口文档中尚未正式提供的接口需求。  
> 当前可先通过已有接口或本地缓存兜底，但为了后续功能完整性和数据一致性，建议后端逐步补齐。

## 1. 当前登录管理员信息

### 1.1 背景

当前管理端左下角需要展示“当前登录用户”的基础信息，例如：

- 姓名
- 账号
- 角色
- 用户 ID
- 可选：最近登录时间

目前 [后端已实现接口文档.md](/Users/lijun/develop/projects/邓总毕设/网吧管理系统/netcafe-platform/doc/后端已实现接口文档.md) 中只有 `POST /auth/login` 会在登录成功时返回：

```json
{
  "admin": {
    "id": 1,
    "username": "admin",
    "name": "张三",
    "role": "SUPER_ADMIN"
  }
}
```

前端现在可以先依赖登录响应并缓存到本地完成展示，但仍存在以下问题：

- 页面刷新后只能依赖本地缓存，无法主动校验数据是否被后端修改
- 若管理员姓名或角色被后端变更，前端无法实时获取最新信息
- 无法作为后续“个人中心”“顶部用户信息”“权限校验补强”的统一入口

### 1.2 需求接口

- **接口**：`GET /auth/me`
- **描述**：获取当前登录管理员的基础信息
- **鉴权**：需要

### 1.3 建议响应

```json
{
  "code": 0,
  "message": "OK",
  "data": {
    "id": 1,
    "username": "admin",
    "name": "张三",
    "role": "SUPER_ADMIN",
    "roleLabel": "超级管理员",
    "lastLoginTime": "2026-03-24T22:10:00"
  }
}
```

### 1.4 前端用途

- 左侧边栏显示当前登录用户信息
- 顶部导航或个人信息区域显示用户摘要
- 页面初始化时同步本地缓存，避免展示过期信息

## 2. 后续可扩展方向

以下不是当前强依赖，但建议后端后续预留：

- `PUT /auth/me`：修改当前登录用户个人信息
- `PUT /auth/me/password`：当前登录用户修改自己的密码
- `GET /auth/permissions`：获取当前登录用户的菜单/按钮权限清单
